CSS 三个实例总结

1. CSS 的三种使用方式
- 行内样式：直接写在标签的 style 属性中，优先级最高，适合临时测试。
- 内部样式：写在 HTML 文件中的 <style> 标签里，适合单个页面。
- 外部样式：通过 <link rel="stylesheet" href="..."> 引入外部 CSS 文件，最推荐，便于维护和复用。

2. 常见 CSS 选择器
- 元素选择器：直接选中标签，如 p、h1。
- 类选择器：用 .className 选择一类元素，适合批量设置样式。
- ID 选择器：用 #idName 选择唯一元素，优先级最高。
- 通配符选择器：*，表示选择所有元素。
- 后代选择器：.father .son，选中父元素内部所有后代。
- 子元素选择器：.father > .son，只选中直接子元素。
- 伪类选择器：如 :hover，表示鼠标悬停状态。
- 伪元素选择器：如 ::after，常用于内容补充和特殊样式。

3. 选择器优先级
- 一般规则：ID 选择器 > 类选择器 > 元素选择器 > 通配符选择器。
- 如果同一个样式属性被多个选择器同时设置，优先级高的会覆盖优先级低的。
- 子元素选择器的优先级通常高于后代选择器。

4. 常见 CSS 属性
- color：设置文字颜色。
- background-color：设置背景颜色。
- font-size：设置文字大小。
- font-family：设置字体。
- line-height：设置行高。
- width / height：设置宽高。
- display：控制元素显示方式。

5. display 属性的三种常见值
- block：块级元素，独占一行，可以设置宽高。
- inline：行内元素，不会独占一行，宽高由内容决定。
- inline-block：兼具块级和行内元素特点，既能设置宽高，又能与其他内容同行显示。

6. 伪类选择器 vs 伪元素选择器
- 伪类选择器：采用 `:hover`、`:active`、`:first-child`，用于选择元素“处于某种状态”或“满足某个位置关系”。
- 伪元素选择器：采用 `::before`、`::after`、`::first-line`，用于创建或选中元素的“虚拟部分”。
- 伪类不生成新内容，只选中符合条件的元素；伪元素可以生成视觉内容或改变元素的子部分样式。
- 伪元素不是 HTML 中真实的 DOM 节点，不能被 JavaScript 直接选中。

7. 伪元素与普通元素的区别
- 普通元素：真实存在于 HTML 文档中，如 `<div>`、`<p>`、`<span>`。
- 伪元素：由 CSS 生成，不是真正的 DOM 元素，例如 `p::before` 在段落前生成一个虚拟内容块。
- 普通元素可以直接写入 HTML，也可以被 JS 访问；伪元素只能在 CSS 中定义，不能直接通过 JS 操作。

8. 浮动（float）
- 定义：float 属性用于控制元素的浮动方向，使元素脱离正常文档流。
- 常见值：
  - left：元素向左浮动
  - right：元素向右浮动
  - none：不浮动（默认值）
  - inherit：继承父元素的浮动属性
- 浮动特性：
  - 浮动元素会脱离正常文档流，向左或向右移动
  - 浮动元素会停靠在容器边缘或其他浮动元素旁边
  - 浮动元素的宽度会根据内容自动收缩
  - 行内元素浮动后会变成块级元素，可以设置宽高
- 浮动的影响：
  - 父元素高度塌陷：父元素无法感知浮动子元素的高度
  - 后续元素会环绕浮动元素排列
- 清除浮动的方法：
  - clear 属性：left（清除左浮动）、right（清除右浮动）、both（清除左右浮动）
  - 伪元素清除法：.clearfix::after { content: ""; display: block; clear: both; }
  - overflow:hidden：触发 BFC（块级格式化上下文）
  - 使用空 div 标签：<div style="clear:both"></div>（不推荐）
- 浮动的应用场景：
  - 多列布局（如两栏、三栏布局）
  - 图文混排（文字环绕图片）
  - 导航栏布局
  - 卡片式布局
- 注意事项：
  - 不要过度使用浮动，现代布局推荐使用 Flexbox 或 Grid
  - 浮动会影响元素的垂直对齐
  - 清除浮动是使用浮动布局的关键

9. 小结
- CSS 的作用是美化网页和控制布局。
- 学习 CSS 时，要重点掌握“选择器、样式属性、优先级和 display 的使用”。
- 了解伪类和伪元素的区别，有助于写出更清晰的样式规则。
- 掌握浮动和清除浮动的方法，是实现复杂布局的基础。
